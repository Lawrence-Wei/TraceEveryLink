2. 产品概述

接线镜（TraceEveryLink）是一个面向机房现场的接线可视化工作台。用户可以通过网页查看机柜、设备、端口和线缆，快速判断某根线从哪里来到哪里去。系统支持手机、Windows 和 Mac 浏览器使用，第一版按照公网部署设计，除登录入口外所有资产、照片、导出和打印能力都需要鉴权访问。

核心价值：
  • 降低现场查线和排障时间。
  • 把接线数据从 Excel/纸质记录升级为可追踪、可复核的系统记录。
  • 通过二维码标签连接物理线缆和数字台账。
  • 通过照片、状态和审计日志提升接线可信度。
  • 为 LLDP/CDP/SNMP 自动发现、变更审批和资产治理打基础。

3. 背景与问题

当前痛点

  • 线缆台账分散在 Excel、纸质记录和个人经验中。
  • 交换机、配线架、墙口、AP、服务器之间缺少统一链路模型。
  • 接线变更后缺乏复核流程，台账容易与真实环境偏离。
  • 线缆标签只能承载有限文字，无法展示完整路径、照片和状态。
  • 公网部署时照片和资产数据不能以静态目录方式裸露。

产品机会

  • 用机柜视图提升现场识别效率。
  • 用端口级模型建立可信台账。
  • 用状态流转区分规划、草稿、待复核、确认、故障和退役。
  • 用二维码和标签打印连接物理世界与系统记录。
  • 用鉴权、MFA、审计和私有照片满足公网部署要求。

用户主要任务典型场景关注点

现场工程师查线、拍照、提交待复核布线、替换线缆、贴标签手机可用、快速定位、操作简单

审计人员查看记录、导出台账资产审查、变更追踪审计日志、数据完整性

角色权限

角色等级权限说明
VIEWER1查看机柜、端口、线缆、照片和二维码。
SURVEYOR2拥有查看权限；可新增线缆、上传照片、更新普通状态。
REVIEWER3拥有现场人员权限；可确认线缆、导出 Excel/PDF、导出标签、批量打印。
ADMIN4拥有全部权限；可删除线缆、配置和删除打印机。

6. 核心流程

登录访问

1. 访问未登录用户访问系统，页面跳转到登录页。
2. 登录用户使用本地账号或 OAuth/OIDC 登录。
3. 会话服务端创建会话并写入 HTTP-only Cookie。
4. 鉴权所有资产、照片、导出、打印 API 校验会话。

现场查线

1. 搜索输入线缆号、设备名、端口名或端点信息。
2. 选择点击线缆列表或机柜端口。
3. 查看右侧展示端口、线缆、路径、照片和二维码。
4. 处理按权限上传照片、改状态、导出标签或打印。

新增与复核

1. 新增SURVEYOR+ 输入线缆编号、标签、颜色和 A/B 端口。
2. 校验系统校验端口存在、端点不同、端口未被活动线缆占用。
3. 待复核现场完成后设置为 pending_verification。
4. 确认REVIEWER+ 设置为 confirmed 并记录复核人。

标签打印

1. 配置ADMIN 配置打印网关。
2. 勾选REVIEWER+ 勾选需要打印的线缆。
3. 发送系统生成 JSON/ZPL/TSPL 并调用网关。
4. 记录打印任务更新为 sent 或 failed。

7. 页面需求

登录页 /login

  • 提供邮箱、密码、MFA/TOTP 输入框。
  • 提供 Cisco SSO、Google、GitHub 登录入口。
  • 本地登录提交到 /api/auth/login。
  • 登录成功进入工作台，登录失败展示错误。
  • 开发环境允许管理员 MFA 关闭，生产环境建议启用。

接线工作台 /

区域元素需求
顶部栏产品名、标题、搜索框、用户角色、刷新、退出常用操作首屏可达，搜索实时过滤线缆。
左侧栏机柜切换、线缆列表、多选框、状态点支持按机柜查看和批量选择线缆。
中央区机柜框、设备卡片、端口网格端口根据线缆状态显示不同样式，点击端口展示详情。
右侧栏新增线缆、导出、打印、详情、二维码、路径、照片、审计根据权限启用/禁用操作，显示最近审计日志。

8. 功能需求

认证与会话

编号需求优先级
AUTH-01支持本地邮箱密码登录。P0
AUTH-02支持可选 TOTP MFA。P0
AUTH-03支持 Cisco、Google、GitHub OAuth/OIDC。P1
AUTH-04未登录访问页面跳转登录页，访问 API 返回 401。P0
AUTH-05登录接口有基础限流，登录失败写审计。P0

库存、端口与线缆

编号需求优先级
INV-01聚合查询机柜、无机柜设备、线缆、打印机和最近审计日志。P0
PORT-01支持 Cisco Catalyst 风格端口名，例如 Gi1/0/24、Te1/1/1。P0
PORT-02配线架前后端口支持 mappedPortId 映射。P0
CABLE-01线缆编号唯一，必须绑定两个不同端口。P0
CABLE-02活动线缆不得重复占用端口，退役线缆不参与占用。P0
CABLE-03确认线缆仅限 REVIEWER 或 ADMIN。P0
CABLE-04删除线缆仅限 ADMIN。P0

线缆状态

状态中文含义用途
planned规划已设计但尚未实施。
draft草稿初始录入，尚未提交复核。
pending_verification待复核现场完成，等待复核确认。
confirmed已确认复核通过，可作为可信台账。
faulty故障线缆或链路存在问题。
retired退役不参与活动端口占用和路径追踪。

照片、导出与打印

编号需求优先级
PHOTO-01照片必须绑定线缆或端口，只允许图片类型，大小不超过 8 MB。P0
PHOTO-02照片存储在私有目录，只能通过鉴权 API 访问。P0
EXPORT-01支持 Excel、PDF、标签 PDF 导出，权限为 REVIEWER+。P0
LABEL-01每根线缆可生成二维码，URL 指向 /?cable={id}。P0
PRINT-01支持 HTTP_JSON、HTTP_ZPL、HTTP_TSPL 打印网关。P0/P1
PRINT-02打印任务记录 queued、sent、failed。P0

审计日志

系统必须记录登录、MFA 失败、OAuth 登录、新增线缆、更新线缆、删除线缆、上传照片、打印任务、打印机配置变更等关键动作。审计日志应包含操作人、动作、实体类型、实体 ID、旧值、新值、IP 和时间。

10. 数据模型

数据库使用 PostgreSQL，ORM 使用 Prisma。核心模型包括用户、账号、会话、机柜、设备、端口、线缆、照片、打印机、打印任务和审计日志。

实体关键字段说明
Useremail, role, mfaEnabled系统用户，本地和 OAuth 用户统一关联。
Accountprovider, providerAccountId登录身份来源，支持 LOCAL、GOOGLE、GITHUB、CISCO。
SessiontokenHash, csrfToken, expiresAt服务端会话和 CSRF Token。
Rackcode, room, heightU机柜。
Devicename, type, uPosition交换机、配线架、墙口、服务器、AP 等。
PortdeviceId, name, mappedPortId设备端口和配线架映射。
CablecableId, status, endpointAPortId, endpointBPortId端口级线缆连接。
Photofilename, sha256, uploadedById私有照片元数据。
LabelPrinterprotocol, endpoint, enabled标签打印机网关配置。
PrintJobprinterId, status, payload打印任务。
AuditLogactorId, action, entityType, oldValue, newValue审计追踪。

关系说明

  • 一个机柜包含多个设备。
  • 一个设备拥有多个端口。
  • 一根线缆连接两个端口。
  • 配线架前后端口通过 mappedPortId 表示逻辑映射。
  • 照片可绑定线缆或端口。
  • 打印任务包含多个打印明细，每个明细关联一根线缆。
  • 审计日志可关联操作用户。

13. 部署运维

本地开发

npm install

本地没有 PostgreSQL 时可启动容器：

docker run --name patchplan-postgres `
  -e POSTGRES_DB=patchplan `
  -e POSTGRES_USER=patchplan `
  -e POSTGRES_PASSWORD=patchplan `
  -p 5432:5432 `
  -d postgres:16

生产部署

服务职责
dbPostgreSQL 数据库。
migrate执行 Prisma schema push 和 seed。
appNext.js 应用。
caddyHTTPS 反向代理，自动证书和安全响应头。
公网部署要求：
  • 只向公网开放 80 和 443。
  • SSH 使用密钥登录，并限制管理员来源 IP。
  • PostgreSQL 不暴露公网端口。
  • 生产环境必须替换默认密码、TOTP secret、数据库密码和 SESSION_SECRET。

备份

scripts/backup.sh 负责创建加密 PostgreSQL dump、加密照片 volume 备份，并自动删除 30 天以前的备份。建议每日执行，每月至少做一次恢复演练，并同步到独立存储。

阶段目标验收
M1 本地可用数据库初始化、管理员登录、种子数据展示、基础查线。类型检查和测试通过，管理员可登录。
M2 现场试用录入真实机柜、照片上传、标签导出、手机浏览器验证。真实线缆可扫码定位，待复核流程跑通。
M3 公网部署Docker Compose、Caddy HTTPS、生产 MFA、备份恢复。业务 API 未登录均拒绝，数据库不暴露公网。
M4 运维增强用户管理、资产导入、打印机管理页、审计检索。管理员可在 UI 内完成日常配置。

待确认问题

  • 是否需要在 UI 中提供用户管理、角色变更、禁用账号能力？
  • 是否需要支持多个机房/站点的权限隔离？
  • 线缆编号规则是否需要强制格式？
  • 标签尺寸是否固定，还是需要按打印机模板配置？
  • 照片是否需要压缩、缩略图和 EXIF 清理？
  • 是否需要从交换机配置或 LLDP/CDP 自动生成待确认线缆？
  • 是否需要和现有 CMDB、工单系统或资产系统同步？
